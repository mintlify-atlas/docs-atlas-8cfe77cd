## Architecture Overview

Zona Jackpot 777 is built on a modern Python web stack optimized for real-time communication:

<Steps>
  <Step title="Flask Web Application">
    The core server runs on Flask 3.1.3, handling HTTP routes, session management, and template rendering. All game logic, user authentication, and API endpoints are managed through Flask.
    
    **Key Components:**
    - Route handlers for each game mode (`/slotmachine`, `/moche`, `/ruleta`)
    - REST API endpoints (`/bet`, `/win`, `/register`)
    - Admin interface for economy management
  </Step>
  
  <Step title="Socket.IO Real-Time Layer">
    Flask-SocketIO 5.3.6 with Eventlet backend provides WebSocket communication for multiplayer games. This enables instant state synchronization across all players in a game room.
    
    **Features:**
    - Room-based multiplayer lobbies
    - Event-driven game state updates
    - Player presence tracking
    - Quick messaging system
  </Step>
  
  <Step title="SQLite Database">
    Lightweight SQLite database (`casino.db`) stores all persistent data with automatic transaction management and rollback safety.
    
    **Schema:**
    ```sql
    usuarios (
      id INTEGER PRIMARY KEY,
      telegram_id TEXT UNIQUE,
      nombre TEXT,
      username TEXT,
      photo_url TEXT,
      bits INTEGER DEFAULT 0,
      total_recargas INTEGER DEFAULT 0,
      total_ganados INTEGER DEFAULT 0
    )
    ```
  </Step>
  
  <Step title="Telegram Bot Bridge">
    Python-telegram-bot 22.6 creates the entry point, launching the Web App from Telegram chat with a simple `/start` command.
    
    **Implementation:**
    ```python
    async def start(update: Update, context: ContextTypes.DEFAULT_TYPE):
        keyboard = [[
            InlineKeyboardButton(
                "🎰 Abrir Zona Jackpot 777", 
                web_app=WebAppInfo(url=config.WEBAPP_URL)
            )
        ]]
        reply_markup = InlineKeyboardMarkup(keyboard)
        await update.message.reply_text(
            "Bienvenido al Zona Jackpot 777 🎰", 
            reply_markup=reply_markup
        )
    ```
  </Step>
</Steps>

## Technology Stack

The platform leverages proven technologies for reliability and performance:

| Component | Technology | Version | Purpose |
|-----------|-----------|---------|----------|
| **Web Framework** | Flask | 3.1.3 | HTTP routing, templating, session management |
| **Real-Time Engine** | Flask-SocketIO | 5.3.6 | WebSocket communication for multiplayer |
| **Async Server** | Eventlet | 0.40.3 | Non-blocking I/O for Socket.IO events |
| **Production Server** | Gunicorn | 25.1.0 | WSGI HTTP server with worker management |
| **Bot Framework** | python-telegram-bot | 22.6 | Telegram Bot API integration |
| **Database** | SQLite | 3.x | Embedded relational database |
| **HTTP Client** | httpx | 0.28.1 | Async HTTP requests (Telegram API calls) |

<Note>
  All dependencies are pinned to specific versions in `requirements.txt` to ensure consistent deployments across environments.
</Note>

## Project Structure

Understanding the codebase organization:

```

├── app.py                 # Main Flask application (347 lines)
```
