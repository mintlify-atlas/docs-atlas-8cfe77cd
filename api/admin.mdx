## Overview

The Admin Panel provides a secure interface for managing users, recharging bits, and monitoring system statistics. It includes user management operations, bit recharge functionality, and leaderboard queries.

<Warning>
  This endpoint requires administrator authentication. The admin password is currently hardcoded as `"admin"` in production (see security considerations below).
</Warning>

## Database Operations

### Recharge Bits

Updates user balance and tracks total recharges.

```python database.py:141-157
def recargar_bits(telegram_id: str, cantidad: int) -> bool:
    """
    Recarga bits a un usuario existente.
    Retorna True si se realizó la recarga.
    """
    if not telegram_id or cantidad <= 0:
        return False
    
    with get_connection() as conn:
        cursor = conn.execute("""
            UPDATE usuarios
            SET bits = bits + ?,
                total_recargas = total_recargas + ?
            WHERE telegram_id = ?
        """, (cantidad, cantidad, telegram_id))
        
        return cursor.rowcount > 0
```

### Get All Users

Retrieves complete user list ordered by bits balance.

```python database.py:95-108
def obtener_todos_usuarios() -> list:
    """
    Obtiene la lista completa de usuarios registrados.
    Retorna una lista de diccionarios con la información de cada usuario.
    """
    with get_connection() as conn:
        rows = conn.execute("""
            SELECT id, telegram_id, nombre, username, photo_url, 
                   bits, total_recargas, total_ganados
            FROM usuarios
            ORDER BY bits DESC, id ASC
        """).fetchall()
        
        return [dict(row) for row in rows]
```

### Top Recharges Leaderboard

Returns top 10 users by total bits recharged.

```python database.py:111-123
def obtener_top_recargas() -> list:
    """
    Obtiene los 10 usuarios que más bits han recargado (vía admin).
    """
    with get_connection() as conn:
        rows = conn.execute("""
            SELECT telegram_id, nombre, username, photo_url, total_recargas
            FROM usuarios
            WHERE total_recargas > 0
            ORDER BY total_recargas DESC
            LIMIT 10
        """).fetchall()
        return [dict(row) for row in rows]
```

### Top Winners Leaderboard


## User Data Structure

Each user object contains:

```typescript
interface Usuario {
  id: number;                    // Auto-increment primary key
  telegram_id: string;           // Unique Telegram identifier
  nombre: string;                // User's display name
  username: string | null;       // Telegram username (@handle)

  bits: number;                  // Current balance (>= 0)

  total_ganados: number;         // Lifetime game winnings
}
```

## Security Considerations

<Warning>
  **Critical Security Issues** that must be addressed in production:
</Warning>

### 1. Hardcoded Password

**Current Implementation** (`app.py:323`):
```python
ADMIN_PASSWORD = "admin"
```

**Recommended Solution**:
```python
import os
ADMIN_PASSWORD = os.environ.get("ADMIN_PASSWORD")

if not ADMIN_PASSWORD:
    raise ValueError("ADMIN_PASSWORD environment variable must be set")
```

### 2. No Rate Limiting

The endpoint has no protection against brute-force password attacks.

**Recommendation**:
- Implement rate limiting (e.g., Flask-Limiter)
- Add account lockout after failed attempts
- Log all authentication attempts

### 3. No HTTPS Enforcement

**Recommendation**:
- Enforce HTTPS for all admin operations
- Use secure cookies for session management
- Implement CSRF protection

### 4. No Session Management

**Recommendation**:
- Implement proper session-based authentication
- Add session timeout
- Use Flask-Login or similar library

### 5. SQL Injection Protection

**Current Status**: ✅ Protected (uses parameterized queries)

All database operations use parameterized queries:
```python
cursor = conn.execute("""
    UPDATE usuarios
    SET bits = bits + ?, total_recargas = total_recargas + ?
    WHERE telegram_id = ?
""", (cantidad, cantidad, telegram_id))
```

### 6. Input Validation

**Current Validation**:
- Client-side: JavaScript pattern matching
- Server-side: Minimal (password check only)

**Recommendation**:
- Add server-side validation for all inputs
- Validate `telegram_id` format
- Validate `cantidad` ranges
- Sanitize all user inputs

## Error Handling

### Invalid User

If `telegram_id` doesn't exist in database:
- `recargar_bits()` returns `False`
- Operation fails silently
- No error message shown to admin

**Recommendation**: Add explicit user existence check and error message.

### Invalid Amount

If `cantidad <= 0`:
- `recargar_bits()` returns `False`
- No error message shown

**Recommendation**: Add validation and error feedback.
