## Overview

The **Bits** system is the virtual currency powering all gameplay in Zona Jackpot 777. Users bet Bits on games, earn winnings, and can receive admin-issued recharges.

## Database Schema

The currency system is stored in the `usuarios` table with the following fields:

```sql
CREATE TABLE usuarios (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    telegram_id TEXT NOT NULL UNIQUE,
    nombre TEXT NOT NULL,
    bits INTEGER NOT NULL DEFAULT 0 CHECK (bits >= 0),
    total_recargas INTEGER DEFAULT 0,
    total_ganados INTEGER DEFAULT 0,
    username TEXT,
    photo_url TEXT
)
```

**Key Columns:**
- `bits` - Current balance (with CHECK constraint preventing negatives)
- `total_recargas` - Lifetime admin recharges
- `total_ganados` - Lifetime winnings from games

<Note>
The `CHECK (bits >= 0)` constraint ensures users cannot have negative balances.
</Note>

## Core Operations

### Getting Balance

Retrieve a user's current Bits (`database.py:78-93`):

```python
def obtener_bits(telegram_id: str) -> int:
    if not telegram_id:
        raise ValueError("telegram_id es obligatorio")

    with get_connection() as conn:
        row = conn.execute("""
            SELECT bits
            FROM usuarios
            WHERE telegram_id = ?
        """, (telegram_id,)).fetchone()

        return row["bits"] if row else 0
```

### Deducting Bits (Betting)

The `/bet` endpoint deducts Bits before gameplay begins (`app.py:246-279`):

```python
@app.route('/bet', methods=["POST"])
def bet():
    if "telegram_id" not in session:
        return jsonify({
            "status": "error",
            "message": "Usuario no autenticado"
        }), 401

    data = request.get_json()
    cantidad = int(data.get("cantidad", 0))

    if cantidad <= 0:
        return jsonify({
            "status": "error",
            "message": "Cantidad inválida"
        }), 400

    telegram_id = session["telegram_id"]

    success = database.descontar_bits(telegram_id, cantidad)
    bits_actuales = database.obtener_bits(telegram_id)

    if success:
        return jsonify({
            "status": "ok",
            "bits": bits_actuales
        })
    else:
        return jsonify({
            "status": "error",
            "message": "Fondos insuficientes",
            "bits": bits_actuales
        }), 400
```

**Database implementation** (`database.py:179-198`):

```python
def descontar_bits(telegram_id: str, cantidad: int) -> bool:
    if not telegram_id:
        return False

    if cantidad <= 0:
        return False

    with get_connection() as conn:
        cursor = conn.execute("""
            UPDATE usuarios
            SET bits = bits - ?
            WHERE telegram_id = ? AND bits >= ?
        """, (cantidad, telegram_id, cantidad))

        return cursor.rowcount > 0
```

<Tip>
The atomic `WHERE bits >= ?` check prevents race conditions and ensures sufficient funds before deduction.
</Tip>

**Database implementation** (`database.py:160-177`):

```python
def registrar_ganancia(telegram_id: str, cantidad: int) -> bool:
    if not telegram_id or cantidad <= 0:
        return False

    with get_connection() as conn:
        cursor = conn.execute("""
            UPDATE usuarios
            SET bits = bits + ?,
                total_ganados = total_ganados + ?
            WHERE telegram_id = ?
        """, (cantidad, cantidad, telegram_id))

        return cursor.rowcount > 0
```

<Note>
Winnings update both `bits` (current balance) and `total_ganados` (lifetime stats) in a single transaction.
</Note>

## Admin Recharge System

### Recharge Endpoint

Administrators can add Bits to any user account via the `/admin` interface (`app.py:313-340`):

```python
@app.route('/admin', methods=["GET", "POST"])
def admin():
    mensaje = ""

    if request.method == "POST":
        telegram_id = request.form["telegram_id"]
        cantidad = int(request.form["cantidad"])
        password = request.form.get("password")

        ADMIN_PASSWORD = "admin"

        if password == ADMIN_PASSWORD:
            database.recargar_bits(telegram_id, cantidad)
            mensaje = f"✅ Éxito: Se recargaron {cantidad} Bits al usuario con ID {telegram_id}"
        else:
            mensaje = "❌ Error: Contraseña de administrador incorrecta."
            
    usuarios = database.obtener_todos_usuarios()
    top_recargas = database.obtener_top_recargas()
    top_ganadores = database.obtener_top_ganadores()

    return render_template("admin.html", 
                         mensaje=mensaje, 
                         usuarios=usuarios, 
                         top_recargas=top_recargas, 
                         top_ganadores=top_ganadores)
```

<Warning>
The default admin password is hardcoded. In production, use environment variables and strong authentication.
</Warning>

### User List Query

Retrieve all users sorted by balance (`database.py:95-109`):

```python
def obtener_todos_usuarios() -> list:
    with get_connection() as conn:
        rows = conn.execute("""
            SELECT id, telegram_id, nombre, username, photo_url, 
                   bits, total_recargas, total_ganados
            FROM usuarios
            ORDER BY bits DESC, id ASC
```

## Context Manager Pattern

All database operations use a context manager for safe connection handling (`database.py:8-24`):

```python
@contextmanager
def get_connection():
    conn = sqlite3.connect(DB_NAME)
    conn.row_factory = sqlite3.Row
    try:
        yield conn
        conn.commit()
    except Exception:
        conn.rollback()
        raise
    finally:
        conn.close()
```

<Tip>
This pattern ensures all transactions are committed on success and rolled back on errors, preventing data corruption.
</Tip>
