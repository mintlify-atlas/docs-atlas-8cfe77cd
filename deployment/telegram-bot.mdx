## Configuring the Bot Token

### Save Token to Configuration

Add your bot token to `config.py`:

```python config.py
BOT_TOKEN = "8511146591:AAGZP46Bs0NaqzQvXeduDdwzvA3Tyd02JsU"
WEBAPP_URL = "https://your-app-name.herokuapp.com"
```

<Warning>
  For production, use environment variables instead of hardcoding:
  
  ```python
  import os
  BOT_TOKEN = os.environ['BOT_TOKEN']
  ```
</Warning>

### How the Bot Uses the Token

The token is used to authenticate with Telegram's Bot API:

```python main.py
import config
from telegram.ext import Application

def main():
    app = Application.builder().token(config.BOT_TOKEN).build()
    app.add_handler(CommandHandler("start", start))
    app.run_polling()
```

## WebApp Configuration

### Setting the WebApp URL

The WebApp URL must:
- Use **HTTPS** (required by Telegram)
- Be publicly accessible
- Match your deployed application URL

```python config.py
WEBAPP_URL = "https://zona-jackpot-777.herokuapp.com"  # Production
# WEBAPP_URL = "https://abc123.ngrok.io"  # Local testing with ngrok
```

<Note>
  For local development, use [ngrok](https://ngrok.com) to create an HTTPS tunnel:
  

  Then use the `https://` URL provided by ngrok.
</Note>

When a user clicks the "🎰 Abrir Zona Jackpot 777" button:

2. Loads your Flask application from `WEBAPP_URL`

## Running the Bot

### Polling Mode (Development)

Run the bot locally for testing:

```bash
python main.py
```

The bot will continuously poll Telegram for new messages:

```python main.py:15
app.run_polling()
```

**Pros:**
- Easy to set up
- No webhook configuration needed
- Works behind firewalls

**Cons:**
- Higher latency
- Must keep process running
- Not ideal for production

### Webhook Mode (Production)

For production, use webhooks instead of polling:

```python main.py
import config
from telegram.ext import Application

def main():
    app = Application.builder().token(config.BOT_TOKEN).build()
    app.add_handler(CommandHandler("start", start))
    
    # Webhook mode
    app.run_webhook(
        listen="0.0.0.0",
        port=8443,
        url_path=config.BOT_TOKEN,
        webhook_url=f"https://yourdomain.com/{config.BOT_TOKEN}"
    )
```

**Pros:**
- Lower latency (instant updates)
- More efficient for production
- Scales better

**Cons:**
- Requires HTTPS public URL
- More complex setup

<Note>
  For Heroku/Render, you may need to run the bot as a separate worker process or use webhook mode integrated with your Flask app.
</Note>

## Bot Token Security

### Protecting Your Token

<Warning>
  If your bot token is compromised, attackers can:
  - Send messages as your bot
  - Access user data
  - Modify bot configuration
  - Delete your bot
</Warning>

**Security best practices:**

<Steps>
  <Step title="Use Environment Variables">
    Never hardcode tokens in source code:
    
    ```python
    import os
    BOT_TOKEN = os.environ['BOT_TOKEN']  # Read from environment
    ```
  </Step>

  <Step title="Add to .gitignore">
    Prevent accidental commits:
    
    ```txt .gitignore
    config.py
    .env
    *.pyc
    __pycache__/
    ```
  </Step>

  <Step title="Rotate Compromised Tokens">
    If your token is exposed:
    
    1. Send `/revoke` to [@BotFather](https://t.me/BotFather)
    2. Select your bot
    3. Get new token
    4. Update your deployment
  </Step>

  <Step title="Restrict Access">
    Only share tokens with:
    - Deployment platform environment variables
    - Trusted team members
    - Secure secret management systems (e.g., HashiCorp Vault)
  </Step>
</Steps>

### Checking for Leaks

Before deploying, verify no tokens are in git history:

```bash
git log -p | grep -i "BOT_TOKEN"
```

If found, rotate the token immediately and clean git history:

```bash
git filter-branch --force --index-filter \
  "git rm --cached --ignore-unmatch config.py" \
  --prune-empty --tag-name-filter cat -- --all
```

## User Registration Flow

When a user first opens your WebApp:

1. Telegram passes user data via `window.Telegram.WebApp`:
   ```javascript
   const user = window.Telegram.WebApp.initDataUnsafe.user;
   ```

2. Your frontend sends registration request:
   ```javascript
   fetch('/register', {
     method: 'POST',
     headers: { 'Content-Type': 'application/json' },
     body: JSON.stringify({
       telegram_id: user.id,
       nombre: `${user.first_name} ${user.last_name || ''}`,
       username: user.username,
       photo_url: user.photo_url
     })
   });
   ```

3. Backend registers user in database:
   ```python app.py:57-77
   @app.route('/register', methods=["POST"])
   def register():
       data = request.get_json()
       telegram_id = str(data.get("telegram_id"))
       nombre = data.get("nombre")
       username = data.get("username", "")
       photo_url = data.get("photo_url", "")
       
       database.agregar_usuario(telegram_id, nombre, username, photo_url)
       session["telegram_id"] = telegram_id
       session["nombre"] = nombre
       
       return jsonify({"status": "ok"})
   ```

## Troubleshooting

### Bot Not Responding

**Check:**
- Is `python main.py` running?
- Is the bot token correct?
- Any errors in console?

```bash
python main.py
# Should show no errors and start polling
```

### WebApp Button Not Working

**Common issues:**

1. **HTTP instead of HTTPS**
   - Telegram requires HTTPS for WebApps
   - Use ngrok for local testing

2. **Wrong URL**
   - Verify `WEBAPP_URL` matches your deployed app
   - Test URL in browser first

3. **CORS Errors**
   - Check browser console for errors
   - Flask-SocketIO is configured with `cors_allowed_origins='*'`

### User Not Registered

**Check:**
- Is the `/register` endpoint being called?
- Check browser network tab for errors
- Verify `telegram_id` is being sent correctly

## Production Deployment

### Deploying with Flask App

Update your `Procfile` to run both processes:

```txt Procfile
web: gunicorn -k eventlet -w 1 app:app
bot: python main.py
```

<Note>
  Heroku free tier only allows one web dyno. You may need to:
  - Use webhook mode instead of polling
  - Deploy bot separately
  - Upgrade to paid tier for multiple processes
</Note>

### Webhook Integration

Alternatively, integrate webhooks into your Flask app:

```python app.py
from telegram import Update
from telegram.ext import Application

# Initialize bot
bot_app = Application.builder().token(config.BOT_TOKEN).build()

@app.route(f'/telegram/{config.BOT_TOKEN}', methods=['POST'])
async def telegram_webhook():
    update = Update.de_json(request.get_json(), bot_app.bot)
    await bot_app.process_update(update)
    return 'OK'
```

Then set webhook:
```bash
curl -X POST https://api.telegram.org/bot<TOKEN>/setWebhook \
  -d url=https://yourdomain.com/telegram/<TOKEN>
```
