## Overview

This quickstart will take you from zero to a fully functional casino platform running locally with test mode enabled. You'll be able to access all game modes and test the complete user experience without needing Telegram authentication.

<Note>
  **Time Required**: ~5 minutes
  
  **Prerequisites**: 
  - Python 3.11+ installed
  - Git installed
  - Terminal/command line access
</Note>

## Quick Start Steps

<Steps>
  <Step title="Clone the Repository">
    First, get the source code onto your local machine:
    
    ```bash
    git clone <repository-url>
    cd zona-jackpot-777
    ```
    
    <Tip>
      If you don't have the repository URL yet, download the source files and extract them to a folder called `zona-jackpot-777`.
    </Tip>
  </Step>
  
  <Step title="Install Dependencies">
    Create a virtual environment and install all required Python packages:
    
    ```bash
    # Create virtual environment
    python -m venv venv
    
    # Activate it (Windows)
    venv\Scripts\activate
    
    # Activate it (macOS/Linux)
    source venv/bin/activate
    
    # Install dependencies
    pip install -r requirements.txt
    ```
    
    This installs Flask, SocketIO, python-telegram-bot, and all other dependencies defined in `requirements.txt`.
    
    <Warning>
      Make sure you're using Python 3.11 or newer. Check with `python --version`.
    </Warning>
  </Step>
  
  <Step title="Initialize the Database">
    The database initializes automatically on first run, but you can pre-initialize it:
    
    ```bash
    python -c "import database; database.init_db()"
    ```
    
    This creates `casino.db` with the `usuarios` table structure:
    
    ```sql
    CREATE TABLE usuarios (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        telegram_id TEXT NOT NULL UNIQUE,
        nombre TEXT NOT NULL,
        bits INTEGER NOT NULL DEFAULT 0 CHECK (bits >= 0),
        username TEXT,
        total_recargas INTEGER DEFAULT 0,
        total_ganados INTEGER DEFAULT 0,
        photo_url TEXT
    );
    ```
    
    <Note>
      The database file `casino.db` will be created in the project root directory.
    </Note>
  </Step>
  
  <Step title="Run the Application">
    Start the Flask development server:
    
    ```bash
    python app.py
    ```
    
    You should see output like:
    
    ```
    * Serving Flask app 'app'
    * Debug mode: off
    WARNING: This is a development server. Do not use it in a production deployment.
    * Running on all addresses (0.0.0.0)
    * Running on http://127.0.0.1:10000
    * Running on http://192.168.1.x:10000
    ```
    
    The app listens on port `10000` by default (configurable via `PORT` environment variable).
  </Step>
  
  <Step title="Access the Application">
    Open your browser and navigate to:
    
    ```
    http://localhost:10000
    ```
    
    **You'll automatically be logged in as a test user** with these credentials:
    
    - **Telegram ID**: `12345`
    - **Name**: `Usuario de Prueba`
    - **Username**: `tester`
    - **Starting Bits**: `500`
    
    This is handled by the test mode in `app.py:34-52`:
    
    ```python
    # TEST MODE: Provide a test user with bits when running locally
    test_id = "12345"
    test_nombre = "Usuario de Prueba"
    
    # Add test user to database if it doesn't exist
    database.agregar_usuario(test_id, test_nombre, "tester")
    
    # Give them 500 bits if they have less
    bits = database.obtener_bits(test_id)
    if bits < 500:
        database.recargar_bits(test_id, 500)
        bits = database.obtener_bits(test_id)
    ```
  </Step>
</Steps>

### Testing Each Game Mode

<Accordion title="Slot Machine">
  Click on the Slot Machine card to access `http://localhost:10000/slotmachine`
  
  **What to try:**
  1. Select your bet amount (10, 25, 50, or 100 Bits)
  2. Click "SPIN" to start the reels
  3. Watch for matching symbols:
     - **3 matching symbols**: 10x multiplier
     - **2 matching symbols**: 2x multiplier
     - **Cherry match**: 5x multiplier
  4. Your Bits balance updates automatically via `/bet` and `/win` API calls
  
  **Technical flow:**
  ```javascript
  // When you click SPIN
  POST /bet {cantidad: betAmount}
  // Deducts Bits from your balance
  
  // If you win
  POST /win {cantidad: winAmount}
  // Adds winnings to your balance
  ```
</Accordion>

<Accordion title="Moche (Multiplayer)">
  Click on Moche to access `http://localhost:10000/moche`
  
  **What to try:**
  1. Click "Create Room" button
  2. Configure room settings:
     - **Difficulty**: Easy (50 Bits) / Medium (150) / Hard (350) / Pro (700)
     - **Visibility**: Public (anyone can join) or Private (invite only)
     - **Player Slots**: 2-4 total players
  3. Click "Ready" to signal you're prepared
  4. As host, click "Start Game"
  5. AI bots automatically fill empty slots based on difficulty
  
  **Room creation API:**
  ```python
  POST /api/rooms
  {
    "is_private": false,
    "difficulty": "easy",
    "total_slots": 4
  }
  # Returns: {"status": "ok", "room_id": "a3f9c2d1"}
  ```
  
  **Socket.IO Events:**
  - `join_moche_room`: Join an existing room
  - `toggle_ready`: Mark yourself as ready
  - `start_moche_game`: Host initiates game start
  - `room_update`: Broadcast when players join/leave
  - `game_started`: Signals game has begun
</Accordion>

<Accordion title="Ruleta Francesa">
  Click on Ruleta to access `http://localhost:10000/ruleta`
  
  **What to try:**
  1. Place chips on the betting table
  2. Available bet types:
     - **Straight up**: Single number (35:1 payout)
     - **Split**: Two adjacent numbers (17:1)
     - **Street**: Row of 3 numbers (11:1)
     - **Corner**: 4 numbers (8:1)
     - **Red/Black**: Color bets (1:1)
     - **Even/Odd**: Parity bets (1:1)
     - **Dozens**: 1-12, 13-24, 25-36 (2:1)
  3. Click "SPIN" to launch the ball
  4. Winnings calculated based on where the ball lands
  
  **Betting flow:**
  ```javascript
  // Total all chip placements
  totalBet = chips.reduce((sum, chip) => sum + chip.value, 0)
  
  // Deduct from balance
  POST /bet {cantidad: totalBet}
  
  // Calculate winnings based on landing number
  winAmount = calculatePayouts(landingNumber, chipPlacements)
  
  // Credit account
  POST /win {cantidad: winAmount}
  ```
</Accordion>

## Common Issues & Solutions

<AccordionGroup>
  <Accordion title="Port 10000 already in use">
    **Error**: `OSError: [Errno 48] Address already in use`
    
    **Solution**: Change the port via environment variable:
    
    ```bash
    # macOS/Linux
    PORT=8080 python app.py
    
    # Windows (Command Prompt)
    set PORT=8080 && python app.py
    
    # Windows (PowerShell)
    $env:PORT=8080; python app.py
    ```
    
    Then access at `http://localhost:8080`
  </Accordion>
  
  <Accordion title="Module not found errors">
    **Error**: `ModuleNotFoundError: No module named 'flask'`
    
    **Solution**: Ensure virtual environment is activated and dependencies installed:
    
    ```bash
    # Verify you're in venv
    which python  # Should show path to venv/bin/python
    
    # Reinstall dependencies
    pip install -r requirements.txt
    
    # Verify Flask is installed
    pip show flask
    ```
  </Accordion>
  
  <Accordion title="Database locked errors">
    **Error**: `sqlite3.OperationalError: database is locked`
    
    **Solution**: SQLite doesn't handle high concurrency well. This usually happens if:
    - Multiple processes are accessing the database
    - A previous Python process didn't close properly
    
    Fix:
    ```bash
    # Stop all Python processes
    pkill python  # macOS/Linux
    taskkill /F /IM python.exe  # Windows
    
    # Delete database and recreate
    rm casino.db
    python -c "import database; database.init_db()"
    
    # Restart app
    python app.py
    ```
  </Accordion>
  
  <Accordion title="Bits not updating after bet">
    **Error**: Balance shows same amount after spinning slot machine
    
    **Solution**: Check browser console (F12) for errors. Common causes:
    - Session expired (refresh page)
    - API endpoint returned error (check terminal logs)
    - JavaScript error preventing API call
    
    Manual test:
    ```bash
    # In a new terminal, test the /bet endpoint
    curl -X POST http://localhost:10000/bet \
      -H "Content-Type: application/json" \
      -H "Cookie: session=<your-session-cookie>" \
      -d '{"cantidad": 50}'
    ```
  </Accordion>
</AccordionGroup>
