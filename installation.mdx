## System Requirements

### Minimum Requirements

- **Python**: 3.11.9 or newer (3.11.x recommended)
- **RAM**: 512MB minimum, 1GB recommended
- **Storage**: 100MB for application + dependencies
- **OS**: Linux, macOS, or Windows 10/11

### Required Tools

```bash
# Check Python version (must be 3.11+)
python --version
# Output: Python 3.11.9

# Verify pip is installed
pip --version
# Output: pip 24.x from ...

# Git for cloning repository
git --version
# Output: git version 2.x
```

<Warning>
  The application requires Python 3.11+ due to dependencies in `python-telegram-bot` 22.6. Earlier versions will fail during installation.
</Warning>

## Local Development Setup

<Steps>
  <Step title="Clone and Navigate">
    Get the source code and enter the project directory:
    
    ```bash
    # Clone the repository
    git clone <repository-url>
    cd zona-jackpot-777
    
    # Verify file structure
    ls -la
    # Should show: app.py, main.py, database.py, requirements.txt, etc.
    ```
  </Step>
  
  <Step title="Create Virtual Environment">
    Isolate dependencies using Python's built-in venv:
    
    ```bash
    # Create virtual environment
    python -m venv venv
    
    # Activate on macOS/Linux
    source venv/bin/activate
    
    # Activate on Windows (Command Prompt)
    venv\Scripts\activate.bat
    
    # Activate on Windows (PowerShell)
    venv\Scripts\Activate.ps1
    
    # Verify activation (should show venv path)
    which python  # macOS/Linux
    where python  # Windows
    ```
    
    <Note>
      Your terminal prompt should change to show `(venv)` indicating the virtual environment is active.
    </Note>
  </Step>
  
  <Step title="Install Dependencies">
    Install all required packages from `requirements.txt`:
    
    ```bash
    pip install -r requirements.txt
    ```
    
    This installs 20 packages including:
    
    | Package | Version | Purpose |
    |---------|---------|----------|
    | Flask | 3.1.3 | Web framework |
    | Flask-SocketIO | 5.3.6 | WebSocket support |
    | eventlet | 0.40.3 | Async I/O |
    | gunicorn | 25.1.0 | Production WSGI server |
    | python-telegram-bot | 22.6 | Telegram Bot API |
    
    **Full dependency list from `requirements.txt`:**
    ```txt
    anyio==4.12.1
    blinker==1.9.0
    certifi==2026.2.25
    click==8.3.1
    colorama==0.4.6
    Flask==3.1.3

    python-telegram-bot==22.6
    Werkzeug==3.1.6
    Flask-SocketIO==5.3.6
    eventlet==0.40.3
    ```
  </Step>
  
  <Step title="Configure Application Settings">
    Create or modify `config.py` with your Telegram bot credentials:
    
    ```python config.py
    # Telegram Bot Token from @BotFather
    BOT_TOKEN = "YOUR_BOT_TOKEN_HERE"
    
    # Web App URL (for local dev, use ngrok or localhost)
    WEBAPP_URL = "http://localhost:10000"
    
    # Optional: Secret key for Flask sessions

    ```
    
    <Accordion title="How to get a Telegram Bot Token">
      1. Open Telegram and search for **@BotFather**
      2. Send `/newbot` command
      3. Follow prompts to choose a name and username
      4. BotFather will reply with your token: `8511146591:AAGZP46Bs0NaqzQvXeduDdwzvA3Tyd02JsU`
      5. Copy the token to `config.py`
      
      **Security:** Never commit `config.py` with real tokens to public repositories!
    </Accordion>
    
    <Warning>
      For local development without Telegram integration, you can skip this step. The app will run in test mode with a sample user.
    </Warning>
  </Step>
  
  <Step title="Initialize Database">
    The database auto-initializes on first run, but you can pre-create it:
    
    ```bash
    python -c "import database; database.init_db()"
    ```
    
    This creates `casino.db` with the schema defined in `database.py:26-54`:
    
    ```sql
    CREATE TABLE IF NOT EXISTS usuarios (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        telegram_id TEXT NOT NULL UNIQUE,
        nombre TEXT NOT NULL,
        bits INTEGER NOT NULL DEFAULT 0 CHECK (bits >= 0),
        username TEXT,
        total_recargas INTEGER DEFAULT 0,
        total_ganados INTEGER DEFAULT 0,
        photo_url TEXT
    );
    ```
    
    **Database location**: `./casino.db` (same directory as `app.py`)
  </Step>
  
  <Step title="Run Development Server">
    Start the Flask application:
    
    ```bash
    python app.py
    ```
    
    Expected output:

    - **Localhost**: `http://localhost:10000`

  </Step>

## Production Deployment

### Deployment to Render.com

Render is a modern cloud platform with free tier support and automatic deployments from Git.

<Steps>
  <Step title="Prepare Repository">
    Ensure these files are in your repository root:
    
    ```bash
    # Required files
    ✅ Procfile           # Gunicorn startup command
    ✅ requirements.txt   # Python dependencies
    ✅ runtime.txt        # Python version
    ✅ app.py            # Main application
    ```
    
    **Verify `Procfile` contents:**
    ```procfile Procfile
    web: gunicorn -k eventlet -w 1 app:app
    ```
    
    **Verify `runtime.txt` contents:**
    ```txt runtime.txt
    python-3.11.9
    ```
    
    Push to GitHub/GitLab:
    ```bash
    git add .
    git commit -m "Prepare for Render deployment"
    git push origin main
    ```
  </Step>
  
  <Step title="Create Render Web Service">
    1. Log in to [render.com](https://render.com)
    2. Click **"New +"** → **"Web Service"**
    3. Connect your GitHub/GitLab repository
    4. Configure the service:
       - **Name**: `zona-jackpot-777`
       - **Environment**: `Python 3`
       - **Build Command**: `pip install -r requirements.txt`
       - **Start Command**: `gunicorn -k eventlet -w 1 app:app`
       - **Instance Type**: `Free` (or paid for better performance)
  </Step>
  
  <Step title="Set Environment Variables">
    In Render dashboard, add these environment variables:
    
    | Key | Value | Purpose |
    |-----|-------|----------|
    | `PORT` | `10000` | Port for Gunicorn |
    | `BOT_TOKEN` | Your Telegram token | From @BotFather |
    | `WEBAPP_URL` | Your Render URL | e.g., `https://zona-jackpot-777.onrender.com` |
    | `SECRET_KEY` | Random secure string | Flask session encryption |
    
    **Generate a secure `SECRET_KEY`:**
    ```bash
    python -c "import secrets; print(secrets.token_hex(32))"
    # Output: a3f9c2d1e5b8... (use this value)
    ```
  </Step>
  
  <Step title="Deploy">
    Click **"Create Web Service"** and wait for deployment:
    
    ```bash
    # Render will execute:
    ==> Installing dependencies from requirements.txt
    ==> Building...
    ==> Starting service with: gunicorn -k eventlet -w 1 app:app
    ==> Deploy successful!
    ```
    
    Your app is now live at: `https://zona-jackpot-777.onrender.com`
    
    <Note>
      Free tier may sleep after 15 minutes of inactivity. First request after sleep takes ~30 seconds to wake up.
    </Note>
  </Step>
  
  <Step title="Update Telegram Bot Configuration">
    Update `config.py` with your production URL:
    
    ```python config.py
    BOT_TOKEN = "8511146591:AAGZP46Bs0NaqzQvXeduDdwzvA3Tyd02JsU"
    WEBAPP_URL = "https://zona-jackpot-777.onrender.com"  # Your Render URL
    ```
    
    Push the change:
    ```bash
    git add config.py
    git commit -m "Update WEBAPP_URL to production"
    git push
    ```
    
    Render will auto-deploy the update.
  </Step>
</Steps>

## Telegram Bot Setup

After deploying your web application, configure the Telegram bot to launch it.

<Steps>
  <Step title="Start the Bot Process">
    The bot runs separately from the web app. You have two options:
    
    **Option A: Run locally (for testing)**
    ```bash
    # Ensure config.py has your BOT_TOKEN and production WEBAPP_URL
    python main.py
    
    # Output:
    # Bot is running...
    ```
    
    **Option B: Deploy bot to a separate service**
    
    Create a new Render background worker or Heroku worker:
    
    ```yaml render.yaml
    services:
      - type: web
        name: zona-jackpot-webapp
        env: python
        buildCommand: pip install -r requirements.txt
        startCommand: gunicorn -k eventlet -w 1 app:app
        
      - type: worker
        name: zona-jackpot-bot
        env: python
        buildCommand: pip install -r requirements.txt
        startCommand: python main.py
    ```
  </Step>
  
  <Step title="Test the Bot">
    1. Open Telegram and search for your bot by username (set in @BotFather)
    2. Send `/start` command
    3. You should see:
       ```
       Bienvenido al Zona Jackpot 777 🎰
       [🎰 Abrir Zona Jackpot 777] <- Button
       ```
    4. Click the button to launch the Web App
    
    **The bot code from `main.py:5-10`:**
    ```python
    async def start(update: Update, context: ContextTypes.DEFAULT_TYPE):
        keyboard = [[
            InlineKeyboardButton(
                "🎰 Abrir Zona Jackpot 777", 
                web_app=WebAppInfo(url=config.WEBAPP_URL)
            )
        ]]
        reply_markup = InlineKeyboardMarkup(keyboard)
        await update.message.reply_text(
            "Bienvenido al Zona Jackpot 777 🎰", 
            reply_markup=reply_markup
        )
    ```
  </Step>
  
  <Step title="User Registration Flow">
    When a user clicks the Web App button:
    
    1. Telegram opens the WEBAPP_URL in an embedded browser
    2. Client JavaScript extracts Telegram user data:
       ```javascript
       const user = Telegram.WebApp.initDataUnsafe.user;
       // {id: 123456789, first_name: "John", username: "john_doe", photo_url: "..."}
       ```
    3. Client sends registration request:
       ```javascript
       POST /register
       {
         "telegram_id": "123456789",
         "nombre": "John",
         "username": "john_doe",
         "photo_url": "https://..."
       }
       ```
    4. Server creates user in database (`app.py:57-77`):
       ```python
       database.agregar_usuario(telegram_id, nombre, username, photo_url)
       session["telegram_id"] = telegram_id
       session["nombre"] = nombre
       ```
    5. User is now authenticated and can play games
  </Step>
</Steps>

### Migrate to PostgreSQL (Production)

For high-traffic production deployments, migrate to PostgreSQL:

<Steps>
  <Step title="Install PostgreSQL Driver">
    ```bash
    pip install psycopg2-binary
    ```
    
    Add to `requirements.txt`:
    ```txt
    psycopg2-binary==2.9.9
    ```
  </Step>
  
  <Step title="Modify database.py">
    Replace SQLite connection with PostgreSQL:
    
    ```python database.py
    import psycopg2
    import os
    
    DATABASE_URL = os.environ.get('DATABASE_URL', 'sqlite:///casino.db')
    
    @contextmanager
    def get_connection():
        if DATABASE_URL.startswith('postgres'):
            conn = psycopg2.connect(DATABASE_URL, sslmode='require')
        else:
            conn = sqlite3.connect("casino.db")
            conn.row_factory = sqlite3.Row
        
        try:
            yield conn
            conn.commit()
        except Exception:
            conn.rollback()
            raise
        finally:
            conn.close()
    ```
  </Step>
  
  <Step title="Set DATABASE_URL">
    ```bash
    # Render automatically provides this for PostgreSQL databases
    # Or set manually:
    export DATABASE_URL="postgresql://user:pass@host:5432/dbname"
    ```
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Socket.IO connection failures">
    **Symptom**: Multiplayer rooms don't sync, players can't join
    
    **Diagnosis**:
    ```bash
    # Check browser console (F12)
    # Look for WebSocket errors:
    WebSocket connection to 'wss://...' failed
    ```
    
    **Solutions**:
    1. **CORS issues**: Ensure `cors_allowed_origins='*'` in `app.py:9`
    2. **HTTP vs HTTPS**: If deploying to HTTPS, Socket.IO must use WSS protocol
    3. **Eventlet not installed**: Verify `pip show eventlet`
    4. **Wrong worker count**: Must use `-w 1` with Gunicorn (only 1 worker for Socket.IO)
    
    **Fix Procfile**:
    ```procfile
    # Correct (1 worker)
    web: gunicorn -k eventlet -w 1 app:app
    
    # Wrong (multiple workers break Socket.IO)
    web: gunicorn -k eventlet -w 4 app:app
    ```
  </Accordion>
  
  <Accordion title="Telegram Web App not opening">
    **Symptom**: Clicking bot button does nothing or shows error
    
    **Check**:
    1. **WEBAPP_URL is HTTPS**: Telegram requires HTTPS for Web Apps (except localhost)
    2. **URL is accessible**: Test by opening `WEBAPP_URL` in a browser
    3. **Bot token is correct**: Verify in @BotFather with `/mybots`
    
    **For local testing with HTTPS**:
    ```bash
    # Use ngrok to create HTTPS tunnel
    ngrok http 10000
    
    # Update config.py with ngrok URL
    WEBAPP_URL = "https://abc123.ngrok.io"
    
    # Restart bot
    python main.py
    ```
  </Accordion>
  
  <Accordion title="High memory usage / app crashes">
    **Symptom**: App crashes with memory errors on free tier hosting
    
    **Analysis**:
    ```bash
    # Check memory usage
    ps aux | grep python
    
    # Monitor in real-time
    top -p $(pgrep python)
    ```
    
    **Optimizations**:
    1. **Limit room history**: Clear old rooms from `RoomManager`
       ```python moche_engine.py
       # Add periodic cleanup
       def cleanup_old_rooms(self):
           for room_id, room in list(self.rooms.items()):
               if room["status"] == "finished" and is_old(room):
                   del self.rooms[room_id]
       ```
    
    2. **Database connection pooling**: Close connections promptly
       ```python
       # Already handled by context manager in database.py
       ```
    
    3. **Reduce worker processes**: Already set to 1 in Procfile
  </Accordion>
  
  <Accordion title="Bits balance incorrect after game">
    **Symptom**: Bits deducted but not added back after win
    
    **Debug**:
    ```bash
    # Check app logs for errors
    tail -f app.log
    
    # Or on Render
    # View Logs tab in dashboard
    ```
    
    **Common causes**:
    1. **Transaction rollback**: Exception during `/win` endpoint
    2. **Session expired**: User session lost, `/win` returns 401
    3. **JavaScript error**: Frontend doesn't call `/win` API
    
    **Verify database directly**:
    ```bash
    sqlite3 casino.db "SELECT telegram_id, bits, total_ganados FROM usuarios WHERE telegram_id='12345';"
    ```
    
    **Fix**: Add more error handling in `app.py:284-308`:
    ```python
    @app.route('/win', methods=["POST"])
    def win():
        try:
            if "telegram_id" not in session:
                return jsonify({"status": "error", "message": "Usuario no autenticado"}), 401
            
            data = request.get_json()
            cantidad = int(data.get("cantidad", 0))
            
            if cantidad <= 0:
                return jsonify({"status": "error", "message": "Cantidad inválida"}), 400
            
            telegram_id = session["telegram_id"]
            success = database.registrar_ganancia(telegram_id, cantidad)
            
            if not success:
                return jsonify({"status": "error", "message": "Error al registrar ganancia"}), 500
            
            bits_actuales = database.obtener_bits(telegram_id)
            return jsonify({"status": "ok", "bits": bits_actuales})
            
        except Exception as e:
            print(f"Error in /win: {e}")  # Log to console
            return jsonify({"status": "error", "message": str(e)}), 500
    ```
  </Accordion>
</AccordionGroup>
